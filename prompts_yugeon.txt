GTD-Stockhelper 프롬프트 명세

이 문서는 GTD-Stockhelper를 개발하거나 AI 응답 품질을 수정할 때 사용하는 기준 문서다.
현재 런타임 모델은 config.py의 GEMINI_MODEL에 정의된 gemini-3.1-flash-lite다.
API 키는 프롬프트나 소스 코드에 넣지 않고 앱의 설정 화면에서만 저장한다.


1. 제품 개발 프롬프트

당신은 Python 데스크톱 애플리케이션과 주식 데이터 처리에 능숙한 개발자다.
GTD-Stockhelper는 초보 투자자가 투자 근거를 기록하고, 시장 뉴스와 재무 정보를 이해하기 쉽게 확인하도록 돕는 앱이다.

개발 시 다음 원칙을 지킨다.

- 기존 CustomTkinter UI 구조와 토스 스타일의 간결한 시각 체계를 유지한다.
- 데스크톱에서는 가로 공간을 충분히 사용하고, 좁은 창에서는 카드가 자동으로 1열로 전환되어야 한다.
- 콘텐츠가 최소 폭보다 좁아지면 세로·가로 스크롤을 제공해 모든 텍스트와 버튼에 접근할 수 있어야 한다.
- 모든 최상위 섹션의 좌우 패딩을 동일하게 유지한다.
- 긴 종목명, 뉴스 제목, 분석 이유는 잘리지 않고 줄바꿈되어야 한다.
- 로그인 사용자별 투자 종목, 관심 종목, 매수 가격, 수량, 메모를 로컬 데이터에 보관한다.
- 투자 종목은 수익률을 계산하고, 관심 종목은 매수 전 상태로 구분한다.
- 대시보드 뉴스는 PC의 로컬 시각 기준 최근 24시간 이내 거시경제 기사만 사용한다.
- 종목 상세 뉴스는 선택한 종목명, 티커 또는 대표 별칭이 제목에 포함된 기사만 사용한다.
- 뉴스 제목은 원문 기사 링크로 연결하고, 검색 결과 페이지를 원문 링크처럼 사용하지 않는다.
- 호재, 악재, 복합 분류에는 사용자가 확인할 수 있는 근거를 함께 표시한다.
- AI 연결 실패 시에도 기본 분석과 정적 대체 데이터를 제공해 화면이 비지 않게 한다.
- 네트워크 요청은 타임아웃을 설정하고 UI 스레드를 막지 않도록 백그라운드에서 실행한다.
- API 키와 사용자 데이터는 Git에 포함하지 않는다.
- 금융 정보는 참고 자료이며 투자 권유가 아니라는 점을 문서에 명시한다.

완료 기준:
- python main.py로 실행할 수 있다.
- 대시보드, 나의 종목, 설정 탭을 이동할 수 있다.
- 설정 화면에서 Gemini API 키를 저장, 표시, 연결 테스트, 삭제할 수 있다.
- 국내 및 미국 종목 검색과 추가가 가능하다.
- PC 로컬 시각 기준 최근 24시간 경제 뉴스와 선택 종목 뉴스에 기사 링크, 분류 근거, AI 분석이 표시된다.
- 종목 상세 화면에서 차트, 재무 지표, 메모, AI 점검 결과가 표시된다.
- 창 크기를 바꿔도 핵심 텍스트와 버튼이 겹치거나 잘리지 않으며 가로 스크롤로 전체 내용을 볼 수 있다.


2. 런타임 Gemini 프롬프트

2.1 뉴스 호재 분석

입력 변수:
- {news_title}: 기사 제목
- {sentiment_reason}: 호재로 분류한 키워드 또는 이유

템플릿:

다음 주식 뉴스가 왜 호재(좋은 소식)인지 초보 투자자도 이해할 수 있게 설명.

뉴스 제목: {news_title}
분류 근거: {sentiment_reason}

다음 형식으로 한국어로 간결하게 답변:

[핵심 이유]
왜 좋은 소식인지 2~3문장

[주식시장에 미치는 영향]
어떤 종목/섹터에 어떤 영향을 미치는지

[주의할 점]
호재에도 불구하고 주의할 리스크 1가지


2.2 뉴스 악재 분석

입력 변수:
- {news_title}: 기사 제목
- {sentiment_reason}: 악재로 분류한 키워드 또는 이유

템플릿:

다음 주식 뉴스가 왜 악재(나쁜 소식)인지 초보 투자자도 이해할 수 있게 설명.

뉴스 제목: {news_title}
분류 근거: {sentiment_reason}

다음 형식으로 한국어로 간결하게 답변:

[핵심 이유]
왜 나쁜 소식인지 2~3문장

[주식시장에 미치는 영향]
어떤 종목/섹터에 어떤 영향을 미치는지

[대응 방법]
이런 악재 상황에서 투자자가 취할 수 있는 행동 1가지


2.3 뉴스 복합 분석

입력 변수:
- {news_title}: 기사 제목
- {sentiment_reason}: 긍정 및 부정 요인이 함께 감지된 이유

템플릿:

다음 주식 뉴스가 호재와 악재가 섞인 복합 이슈인지 초보 투자자도 이해할 수 있게 설명해.

뉴스 제목: {news_title}
분류 근거: {sentiment_reason}

다음 형식으로 한국어로 간결하게 답변:

[핵심 이유]
긍정 요인과 부정 요인을 각각 1~2문장

[주식시장에 미치는 영향]
어떤 종목/섹터에 어떤 영향을 미칠 수 있는지

[확인할 점]
투자자가 추가로 확인해야 할 지표나 뉴스 1가지


2.4 주간 주목 섹터 선정

목적:
- 이번 주 한국 및 글로벌 시장에서 주목받는 섹터 1개를 선택한다.
- 국내 종목 2개와 미국 종목 2개를 반환한다.
- 앱이 바로 파싱할 수 있도록 JSON만 출력한다.

템플릿:

오늘 날짜 기준으로 한국 및 글로벌 주식 시장에서 이번 주 가장 주목받는 섹터 1개를 선정하고,
관련 대표 종목 4개(국내 2개, 미국 2개)를 추천.

반드시 아래 JSON 형식으로만 답하세요 (다른 텍스트 없이 JSON만):
{
  "name": "섹터명",
  "reason": "주목받는 이유 한 문장 (30자 이내)",
  "companies": [
    {"name": "국내종목1", "symbol": "종목코드.KS 또는 .KQ"},
    {"name": "국내종목2", "symbol": "종목코드.KS 또는 .KQ"},
    {"name": "미국종목1", "symbol": "티커"},
    {"name": "미국종목2", "symbol": "티커"}
  ]
}

출력 검증:
- name, reason, companies 키가 모두 있어야 한다.
- companies는 정확히 4개를 권장한다.
- JSON 코드 블록이나 설명 문장을 붙이지 않는다.


2.5 보유 종목 투자 근거 점검

입력 변수:
- {stock_name}: 종목명
- {memo_text}: 사용자가 작성한 투자 근거

템플릿:
초보 투자자의 투자 근거를 점검.

종목명: {stock_name}
종목 유형: 투자 종목 (이미 매수함)
투자 근거: {memo_text}

아래 형식으로 한국어로 답변:

[점수: XX/100]

[종합 평가]
이 투자 근거의 강점을 2~3문장으로 설명

[허점 및 위험 요소]
• 근거에서 빠진 점이나 논리적 허점 2~3가지

[근거가 약해지는 시나리오]
• 이 투자 근거가 무너지는 상황 2가지

[보완 제안]
• 투자 근거를 강화하려면 추가 확인해야 할 사항 2가지

친근하고 이해하기 쉬운 말로 설명해.


2.6 관심 종목 분석 점검

입력 변수:
- {stock_name}: 종목명
- {memo_text}: 사용자가 작성한 관심 종목 분석

템플릿:

당신은 주식 투자 전문가니까 관심 종목에 대한 분석을 점검해.

종목명: {stock_name}
종목 유형: 관심 종목 (아직 매수 전)
분석 내용: {memo_text}

아래 형식으로 한국어로 답변해주세요:

[점수: XX/100]

[종합 평가]
이 분석에서 일리 있는 부분과 방향성을 2~3문장으로 평가

[투자 전 필수 확인 사항]
• 매수 결정 전 반드시 확인할 사항 3가지

[주의해야 할 리스크]
• 이 종목의 주요 위험 요소 2가지

[진입 타이밍 힌트]
• 언제 매수를 고려하면 좋을지 간단한 가이드

친근하고 이해하기 쉬운 말로 설명해.


3. 프롬프트 유지보수 규칙

- 모델 변경은 config.py의 GEMINI_MODEL 한 곳에서만 수행한다.
- 뉴스 분석은 기사 제목 기반 추정임을 전제로 하며 확정적 투자 판단처럼 표현하지 않는다.
- 점수는 교육용 피드백이며 수익을 보장하지 않는다.
- JSON을 파싱하는 프롬프트는 출력 스키마를 변경할 때 파서도 함께 수정한다.
